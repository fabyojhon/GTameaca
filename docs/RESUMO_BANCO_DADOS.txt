⚙️ COMO FUNCIONA

1. User digita parecer em textarea
   └─ event: 'change' → blur → salvarPareceresLocal()
   └─ Chama: db.salvarPareceres(atendId, pareceres)
   └─ Salva: pareceres_[ID] em localStorage
   └─ Marca: respondido = false (temporário)
   └─ Console: "💾 Auto-save: [setorId]"

2. A cada 30 segundos (Timer)
   └─ Função: db.criarBackup()
   └─ Cria: backup_[ID]_[TIMESTAMP]
   └─ Contém: atendimento + matriz + pareceres
   └─ Deleta: Backups antigos (manter últimos 5)

3. User volta para identificacao.html
   └─ sessionStorage: mantém atendimentoId
   └─ localStorage: mantém atendimento_[ID]
   └─ localStorage: mantém pareceres_[ID]
   └─ Tudo intacto!

4. User volta para manifestacoes.html
   └─ Função: carregar()
   └─ Chama: db.obterPareceres(atendId)
   └─ Restaura: pareceres anteriores
   └─ Console: "✅ Pareceres restaurados"
   └─ UI: Pre-filled com dados existentes

5. User envia parecer
   └─ Marca: respondido = true
   └─ Call: db.salvarPareceres()
   └─ Salva: pareceres final
   └─ Cria: backup_[ID]_[TIMESTAMP] permanente
   └─ Alert: "✓ Parecer enviado com sucesso!"

🔄 FLUXO DE PERSISTÊNCIA

entrada → Auto-Save ├─ Local (UI atual)
                    ├─ 30s Timer
                    └─ localStorage backup

                         ↓
                   
localStorage possui 3 tipos:

├─ atendimento_[ID] → Dados básicos
├─ matrizResultado_[ID] → Score + classificação
├─ pareceres_[ID] → Pareceres com metadata + auto-save
└─ backup_[ID]_[TIMESTAMP] → Snapshots completos

                         ↓
                   
User navega para outra página

                         ↓
                   
Volta para manifestacoes.html

                         ↓
                   
carregar() restaura: db.obterPareceres(atendId)

                         ↓
                   
UI pre-filled com dados salvos ✅
